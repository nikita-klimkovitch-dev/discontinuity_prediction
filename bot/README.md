
Запуск проекта
1. устанавливаем Docker https://docs.docker.com/get-docker/
2. клонируем данный репозиторий + делаем запрос на 2 недостующие большие папки у автора бота (в данных папках хранится архитектура и веса некоторых нейронных моделей).
3. файл config ini настраиваем смостоятельно
    -> APP_ID и API_HASH для telethon: https://my.telegram.org/auth
    -> BOT_TOKEN: Write on telegram to @BotFather and follow the instructions
3. открываем консоль в папке bot и пишим команды ниже
4. для создания Docker image:
    `build --tag dockerbot . `
5. для запуска контейнера:
    `docker-compose up`
6. при возникновении ошибок или ненадобности бота рекумендуется не забыть прописать команду 
    `docker-compose down`

Описание бота:
сам бот: @test_disc_prediction_bot
Часть ML: бот определяет по трем заданным контрольным перемещениям контура выработки (глубина 700 м) зоны нарушения сплошности среды (т.е. зоны, в которых с наибольшей вероятностью может произойти разрушение горной породы, что может привести к значительному уменьшению срока эксплуатации горной выработки).
If-else блоки: бот помогает определить методы крепления образовавшихся зон нарушения сплошности среды (пока реализовано криво).
подробнее об ML части:
Шаг 1. по трем заданным перемещениям контура выработки определяется количество зон нарушения сплошности среды (1 или 2). это делается с помощью одного дерева решений. 
Шаг 2_1. если количество зон нарушения сплошности среды равняется 1, то по трем перемещениям определяется 3 параметра эллиптической зоны нарушения сплошности (порядковый id (имеет смысл!!! он идет последовательно по контуру выработки), малая полуось эллипса a, большая полуось b). это делается с помощью одного дерева решений и одной полносвязной нейронной сети.
Шаг 2_2. если количество зон нарушения сплошности среды равняется 2, то по трем перемещениям определяется 2*3 параметров двух эллиптических зон нарушения сплошности. данной действо реализовать было сложнее, поэтому здесь последовательно определяется каждый из неизвестных параметров: сначала по трем параметрам определяется один, затем по четырем параметрам определяется еще один и т.д. для этой цепочки было задействовано 4 дерева решений и 2 полносвязных нейронных сети.


Шаг 1 выполняется при вызове метода predict_num_zones() класса ZonePredictator (файл predictor.py)
Шаг 2 выполняется в if-else блоке метода get_real_zones() класса ZonePredictator (файл predictor.py)
Вывод картинки с рекомендуемыми методами крепления зон нарушения сплошности выполняется при вызове функции plot_zones() (файл visualizer.py)

все файлы, которые начинаются на model - обученные деревья решений или полносвязные нейронные сети (нужны в predictor.py)
все файлы, которые начинаются на scaler - StandartScaler, необходимые нам для скалирования данных в и из деревьев решений или полносвязных нейронных сетей (нужны в predictor.py)
файл dict.csv - словарь для перевода id центра зоны нарушения сплошности (который располагается на контуре выработки) в координаты x и y на плоскости (нужен в visualizer.py)
файл bot.py - основной файл с инструкциями для бота
файл requirements.txt - файл с версиями необходимых для работы бота библиотек
файл Dockerfile - инструкции по сборке контейнера для Docker
файл docker-compose.yml - инструкции по сборке Docker image для Docker
файл config.ini - файл с Вашими данными как телеграмм разработчика и BOT_TOKEN
